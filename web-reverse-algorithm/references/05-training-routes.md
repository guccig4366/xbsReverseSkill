# 训练路线与案例使用法

## 一、训练阶段

### 第一阶段：标准签名回溯能力

建议顺序：

1. 财联社
2. 淘宝 H5
3. 有道翻译
4. 百度翻译

目标：

- 会从最终字段回溯入口
- 会恢复原始串
- 会比对浏览器和本地值

### 第二阶段：加密分层能力

建议顺序：

1. 百度翻译
2. 网易云音乐
3. 响应解密类题

目标：

- 区分明文层、摘要层、加密层、编码层
- 不把所有步骤都混成一个 sign

### 第三阶段：复杂纯算能力

建议顺序：

1. 小红书 `x-s`
2. 抖音 `a_bogus`
3. 滴滴多参数题
4. Spiderdemo T5 / T6

目标：

- 从 writer 倒推中间结构
- 学会插桩和中间数组恢复
- 处理时间戳、随机数和环境依赖

### 第四阶段：验证码 / challenge / 风控

建议顺序：

1. 网易盾
2. 阿里 v2
3. 百度旋转 / 拼多多
4. 极验 3 / 4
5. 腾讯滑块
6. hCaptcha / Cloudflare / PerimeterX

目标：

- 拆 challenge、图像、参数、环境、verify 五条线
- 分清纯算问题、图像问题、环境问题
- 学会 solver 化组织

## 二、同站点多文章的正确读法

不要只保留一个“最终答案”，而要记录思路差异：

- 有的文章先讲入口定位
- 有的文章先讲 AST 解混
- 有的文章先讲 JSVMP 结构
- 有的文章先讲环境位串和补环境
- 有的文章先讲算法骨架和工程接口

这对小红书、a_bogus、腾讯滑块尤其重要。

## 三、案例最少应沉淀什么

每个案例至少留下：

1. 最终请求或最终 sink
2. 请求链闭环
3. 入口定位路径
4. 中间检查点
5. 本地代码骨架
6. 易错点

## 四、如何带新人

推荐讲法：

1. 标准签名题：讲最终字段、原始串、标准算法
2. 混合加密题：讲分层和输入边界
3. JSVMP 题：讲 writer、builder、中间结构恢复
4. 验证码题：讲 challenge、参数、环境、verify 分线

## 五、视频资料最该吸收什么

吸收的不是逐字内容，而是教学顺序：

- 先讲题型价值和目标字段
- 再讲入口定位
- 再讲标准算法层
- 再讲中间数组 / payload / builder
- 最后讲编码层、环境层或 verify 层

## 六、扩库时优先收哪些材料

优先收：

1. 明确写入口定位的
2. 明确写纯算插桩的
3. 明确写 VMP / JSVMP / 转序 / 魔改编码的
4. 明确写补环境 / 原型链 / 反混淆的
5. 明确把研究结果做成项目或 SDK 的
